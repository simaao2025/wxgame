# Godot 4.x 转微信小游戏实现方式研究

本项目基于 Godot 4.x 的 Web 导出版本，通过适配层和自定义加载器实现了在微信小游戏环境下的运行。

## 1. 核心技术路径
**Godot (引擎) -> Web (导出) -> WeChat Mini Game (适配运行)**

- **Godot Web 导出**: 利用 Godot 引擎自带的 Web 导出功能，生成 `.js` 胶水代码和 `.wasm` 二进制文件。
- **适配层 (Adapter)**: 使用 `weapp-adapter.js` 模拟浏览器环境（提供 `window`, `document`, `canvas` 等对象），使 Web 版本的 Godot 能在微信环境中运行。
- **SDK 桥接**: 通过 `godot-sdk.js` 将 Godot 的 `JavaScriptBridge` 与微信原生 API（如文件系统、音频、网络请求等）进行对接。

## 2. 目录结构与关键文件
- **`game.js` (根目录)**: 小游戏入口文件，初始化环境并启动加载器。
- **`godot-loader.js`**: 自定义加载器，负责渲染加载界面（WebGL 实现）、下载分包以及初始化引擎。
- **`engine/` (分包)**: 核心引擎分包，减少首屏加载压力。
  - **`godot.js`**: Godot 引擎的 JavaScript 胶水代码。
  - **`godot.wasm.br`**: 经过 Brotli 压缩的 WebAssembly 引擎核心。
  - **`godot-sdk.js`**: 微信 SDK 适配插件。
  - **`demo-pck.bin`**: 游戏资源包（实际为 Godot 的 `.pck` 文件，重命名为 `.bin` 以适配微信分包策略）。
  - **`game.js` (engine 目录)**: 分包启动逻辑，配置引擎参数并调用 `GODOTSDK.startGame`。

## 3. 初始化流程
1.  **环境注入**: `game.js` 引入 `weapp-adapter.js` 补全浏览器 API。
2.  **加载器启动**: 创建 `GodotLoader` 实例，在主 `canvas` 上利用 WebGL 绘制加载进度条和背景。
3.  **分包下载**: `GodotLoader` 调用 `wx.loadSubpackage` 下载名为 `engine` 的分包。
4.  **引擎启动**:
    - 分包下载完成后，执行 `engine/game.js`。
    - `engine/game.js` 依次引入 `godot-sdk.js` 和 `godot.js`。
    - 调用 `GODOTSDK.startGame(exe, pack)`，其中 `exe` 指向 `godot.js`，`pack` 指向资源包。
5.  **资源加载**: 引擎通过适配后的文件系统读取 `.bin` (PCK) 文件，解析场景并进入游戏。

## 4. 关键适配点
- **文件系统**: 优化了 PCK 的加载方式。支持将 PCK 保持在内存中以提高访问速度，同时也提供了卸载接口以控制内存占用。
- **音频系统**: 针对 iOS 高性能模式进行了优化。建议长音频使用 `sample` 模式，通过微信原生的 `innerAudioContext` 播放，避免帧率波动导致的音频断续。
- **渲染性能**:
  - 启用了 iOS 的 **高性能模式 (High Performance)**。
  - 在 `godot-loader.js` 中对 iOS 高分屏的 `devicePixelRatio` 进行了限制（最大为 2），平衡画质与性能。
- **WASM 优化**: 支持 `godot.wasm.br` 压缩文件，加快下载速度。

## 5. 开发建议
- **资源打包**: 自己的游戏资源需打包成 `.pck` 并更名为 `.bin` 放入 `engine/` 目录。
- **内存管理**: 多个 PCK 加载时，需注意及时通过 `JavaScriptBridge` 调用 `releasePck` 释放内存。
- **工具版本**: 推荐使用较新版本的微信开发者工具（如 RC 2.01.2510241），并开启 Experimental WebAssembly 实验性设置。
